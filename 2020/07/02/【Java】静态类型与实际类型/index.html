相关解释

在本段代码中Father f = new Son()实例化时Father为静态类型，Son为实际类型，并且静态类型在编译期间就是可知的，而实际类型要等到运行期间才可知，Son（）中的doIt（）方法重写了父类中的此方法，而最后执行结果是调用的子类的方法，这说明方法的重写是动态的，以此类推当f作为参数传入时是作为father这个静态类型传入的，因为结果调用的father的方法，这也说明了重载是静态的的
